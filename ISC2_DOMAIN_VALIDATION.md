## ISC2 CISSP Examination Outline - 8 Domains (Official)

### Domain 1: Security & Risk Management
- **Exam Weight:** 15%
- **Key Topics:** Risk assessment, quantification, governance, compliance frameworks (COBIT, ISO, NIST), policy development
- **Lesson Coverage:** Lesson 1 foundation + Management framework
- **Exam Pitfalls:** Confusing risk tolerance vs. risk appetite

### Domain 2: Asset Security
- **Exam Weight:** 10%
- **Key Topics:** Data classification, handling, retention, lifecycle management, secure disposal
- **Lesson Coverage:** Referenced in Lesson 3 (balancing security controls)
- **Exam Pitfalls:** Thinking classification = sensitivity only

### Domain 3: Security Architecture & Engineering
- **Exam Weight:** 12%
- **Key Topics:** Secure architecture design, cryptographic solutions, **CRYPTOGRAPHY**, system design, control frameworks
- **Lesson Coverage:** Lesson 6 (weakness remediation), Management framework
- **Exam Pitfalls:** Memorizing algorithms instead of understanding principles

### Domain 4: Communication & Network Security
- **Exam Weight:** 12%
- **Key Topics:** Network design, protocols (TLS, IPSec), remote access, VoIP security, network architecture
- **Lesson Coverage:** Lesson 3 (real-world tradeoffs in network controls)
- **Exam Pitfalls:** Confusing Layer 2/3/4 controls with Layer 7

### Domain 5: Identity & Access Management
- **Exam Weight:** 13%
- **Key Topics:** Authentication methods, authorization models (RBAC, ABAC), provisioning/deprovisioning, identity governance
- **Lesson Coverage:** Lesson 5 (practice exam questions on IAM), Lesson 8 (non-tech background perspective)
- **Exam Pitfalls:** Thinking IAM = authentication + password policy only

### Domain 6: Security Assessment & Testing
- **Exam Weight:** 12%
- **Key Topics:** Vulnerability assessment, penetration testing, security testing methodologies, audit, compliance testing
- **Lesson Coverage:** Lesson 6 (systematic remediation mirroring pentesting approach), Lesson 5 (practice exams)
- **Exam Pitfalls:** Confusing vulnerability scanning with penetration testing

### Domain 7: Security Operations
- **Exam Weight:** 15%
- **Key Topics:** Incident response, disaster recovery, business continuity, logging & monitoring, forensics
- **Lesson Coverage:** Lesson 6 (continuous vulnerability scanning cycle), Lesson 10 (consistency in operations)
- **Exam Pitfalls:** Thinking incident response = detect, respond, recover only

### Domain 8: Software Development Security
- **Exam Weight:** 11%
- **Key Topics:** Secure SDLC, secure coding practices, code review, secure deployment, application security testing, supply chain security
- **Lesson Coverage:** Lesson 9 (continuous learning after exam), Management framework (application of security principles)
- **Exam Pitfalls:** Overlooking entirely (commonly skipped by non-developers)
